## [0.8.38] - 2026-05-15

### 變更

- **網站上的更新指引更加清晰。** 首頁和安裝頁面現在會顯示 `deepseek update`，同時保留 Homebrew、npm 和 Cargo 安裝的套件管理器更新路徑。
- **README 設定文件已更新至最新。** 英文、簡體中文和日文 README 現在使用最新的 Docker volume/workspace 呼叫方式，記錄更新路徑，列出目前的提供者/模型切換介面，並將特定版本的功能說明引導回變更日誌。

### 修復

- **OpenAI 相容提供者接收更嚴格的請求主體。** Fireworks 請求現在使用 `reasoning_effort`，不再包含 DeepSeek/Anthropic 風格的頂層 `thinking` 欄位，且聊天工具 schema 不再包含僅限 Anthropic 的中繼資料，如 `allowed_callers`、`defer_loading` 或 `input_examples`（#1592）。
- **pnpm 全域安裝在可選的 postinstall 中不再卡住。** pnpm postinstall 現在跳過安裝時的二元檔下載，留待現有的執行時期下載器在首次執行時驗證或獲取二元檔（#1637）。
- **終端機模式在 TUI 提前退出時恢復。** 清理守衛現在會在啟動後提前返回時恢復原始模式、替代螢幕、焦點事件、滑鼠捕捉、括號貼上、鍵盤旗標和游標可見性（#1593、#1582）。
- **換行的 OSC 8 連結保留其完整目標。** 長的可點擊 URL 現在在每個換行的視覺區塊上重新開啟原始完整連結目標，而不是暴露截斷的超連結目標（#1577）。
- **提供者選擇的模型在啟動和選擇器重新選擇時保持不變。** `/model` 選擇器現在在可用時使用即時提供者模型目錄，已儲存的預設提供者會在首次請求前同步到執行時期設定，且從選擇器重新選擇當前提供者時會保留目前模型，而不是退回提供者預設值（#1632）。
- **OpenAI 相容批次工具呼叫保留所有啟動事件。** 在一個助理訊息中包含多個 `tool_calls` 的串流回應現在保留每個工具使用區塊，而不是將多個工具結果僅與最後一個工具啟動事件配對（#1686）。
- **診斷工具 schema 包含空的 `required` 列表。** 內建 `diagnostics` 工具現在發送 `required: []` 及其空物件 schema，使 DeepSeek 不再將其作為 null required 陣列拒絕（#1685）。
- **Windows 滾輪作為方向鍵捲動在滑鼠捕捉啟用時正常工作。** `composer_arrows_scroll` 現在在 Windows 終端機上預設開啟，即使滑鼠捕捉已啟用，使作為方向鍵到達的滾輪事件捲動對話記錄而不是循環編輯器歷史記錄（#1578）。
- **純 Windows PowerShell / ConHost 使用更平靜的渲染。** 未標記的舊版 Windows 控制台主機現在自動啟用低動態渲染，停用花俏動畫，並將 `synchronized_output = "auto"` 解析為關閉，使串流重繪不會重疊或明顯閃爍（#1590）。
- **LoopGuard 區塊現在計為失敗的工具呼叫。** 相同的工具呼叫區塊現在返回失敗的工具結果而非成功，因此重複被封鎖的檢查清單/工具重試可以觸發現有的失敗警告並停止路徑，而不是無限期地循環（#1574）。
- **拒絕的工具審批範圍限定在精確呼叫。** 拒絕一個寫入/shell 審批現在快取規範的參數指紋，而不是有損的工具/前綴鍵，因此稍後對同一工具以不同參數的呼叫仍然可以被審查和批准（#1617）。

## [0.8.37] - 2026-05-14

### 新增

- **騰訊 Lighthouse + 飛書/Lark 橋接器設定。** 新增了 `/opt/whalebro` Lighthouse runbook、systemd 部署資源、長連接飛書/Lark 橋接器、橋接器設定驗證器，以及用於執行時期、Node、二元檔、環境、systemd 和 localhost 健康檢查的 VPS doctor。
- **騰訊雲端遠端優先入門。** 記錄了 CNB + Lighthouse + 飛書/Lark + 可選 EdgeOne 教學路徑，並為未來的 Lighthouse 部署按鈕新增了非活躍 CNB 部署範本。飛書/Lighthouse 分支現在鏡像到 CNB 以實現騰訊優先引導。
- **Homebrew tap 自動化以發佈為門檻。** 發佈工作流程可以在設定 tap token 時從校驗和清單更新 `Hmbown/homebrew-deepseek-tui`，並在沒有 tap token 時在下載發佈資源之前乾淨地跳過。

### 變更

- **Bing 是預設的 `web_search` 後端。** DuckDuckGo 仍可透過 `[search] provider = "duckduckgo"` 選擇，並保留其 Bing 回退路徑。

### 修復

- **首次執行入門在沒有 API 金鑰的情況下仍可使用。** 缺少金鑰的啟動不再在入門收集提供者設定之前中止 TUI。
- **可串流 HTTP MCP 會話保留其伺服器發出的會話 ID。** 自訂標頭也應用於 GET 預檢請求，修復需要兩者的已驗證 MCP 伺服器。
- **DeepSeek 模型補全使用標準 ID。** 別名補全現在在寫入設定之前解析為穩定的 DeepSeek 模型名稱。
- **終端機和子處理程序可靠性更緊密。** 訊號關閉現在恢復終端機，子任務保留代理環境變數，Windows Enter / CSI-u 輸入處理避免了先前的事件不匹配。
- **長終端機文字換行而不是溢出。** 串流輸出、差異渲染和分頁器現在對過長無空格和 CJK 連續文字進行強制換行。
- **發佈和平台邊緣情況更安全。** TUI 不再觸發 Windows Instant-underflow 測試路徑，不受支援的桌面目標編譯外部 URL 開啟器，舊版 DeepSeek CN 提供者別名反序列化為標準 DeepSeek 提供者。
- **頁尾診斷更少晦澀。** 前綴快取穩定性不再顯示在預設頁尾中，選擇加入的 `/statusline` 晶片現在顯示 `cache prefix 100%` 而不是模糊的 `P 100%`。
- **飛書/Lark 橋接器相依套件安裝已鎖定和審計。** 橋接器現在附帶套件鎖定，在 Lighthouse 上可用時使用 `npm ci` 安裝，並將 Lark SDK 的傳遞 `axios` 相依覆蓋為已修補行。
- **中國友好更新回退。** `deepseek update` 現在透過 `DEEPSEEK_TUI_RELEASE_BASE_URL` 和 `DEEPSEEK_TUI_VERSION` 支援鏡像發佈資源，其網路失敗提示將 GitHub 被封鎖網路後的使用者引導至兩個已發佈二元檔的 CNB `cargo install --git` 路徑。
- **CNB 是預設的騰訊發佈候選鏡像。** CNB 同步工作流程現在鏡像飛書/Lighthouse 發佈分支，使騰訊 Lighthouse 引導可以在發佈分支合併之前使用 CNB。

## [0.8.36] - 2026-05-14

### 新增

- **右側邊欄可以隱藏以利於複製友善終端機。** `sidebar_focus = "hidden"`（或目前會話的 `Ctrl+Alt+0`）移除工作/任務/代理/上下文欄，使原始終端機選擇不會將邊欄邊框隨對話記錄文字一起複製。

### 變更

- **子代理完成交接更精簡且更有利於快取。** 內部 `<deepseek:subagent.done>` 哨兵現在指向前面的人類摘要行，而不是在發送給父模型的 JSON 中重複摘要、經過時間和步驟計數。
- **前綴穩定性在快取遙測旁邊預設可見。** 頁尾現在在預設狀態佈局中包含前綴穩定性晶片，當系統/工具前綴本身穩定時，低最後請求快取命中率不再顯示為硬錯誤。
- **RLM 批次輔助工具現在需要明確的獨立性斷言。** `sub_query_batch`、`sub_query_map` 和低階 `*_batched` 輔助工具拒絕相依不安全平行扇出，除非呼叫者傳遞 `dependency_mode="independent"`，而 RLM 現在公開 `sub_query_sequence` 用於 A 到 B 相依工作。

## [0.8.35] - 2026-05-13

一個 0.8.34 後的清理版本，專注於提示衛生、上下文壓力指引，以及保持下一個發佈分支與已發佈的 v0.8.34 標籤明確分離。

> **關於 v0.8.34 貢獻者致謝備註：** Horace Liu（[@liuhq](https://github.com/liuhq)）在 v0.8.34 週期中貢獻了 Nix 套件支援和安裝文件，但不慎從該版本的變更日誌中遺漏。README 貢獻者列表和此備註更正了記錄。

### 變更

- **首輪提示上下文更精簡且更容易審計。** 生成的專案上下文套件現在忽略隱藏工具/快取狀態，在深入之前平衡頂層目錄，且 `/context` 顯示命名的提示層而不是單一不透明的系統 blob。
- **模型可見提示政策已消除衝突。** 基礎和模式提示不再禁止有用的 `deepseek` CLI 診斷，不再要求簡單一步工作的檢查清單，並將長會話壓縮指引對齊約 60% 建議閾值。
- **上下文壓力指引現在有一個分割規則。** 手動 `/compact` 建議在持續工作期間從約 60% 開始，而自動替換壓縮保持在約 80% 附近的選擇加入硬護欄，使 DeepSeek V4 前綴快取經濟性保持完整。
- **Tasks 側邊欄現在會淘汰陳舊的即時工具噪音。** 已完成的活躍工具行短暫停留後離開右側欄；非常舊的執行中 shell 行折疊為單行，而不是佔據整個 Tasks 面板。

### 修復

- **`auto_compact` 設定說明現在報告實際預設值**，自 v0.8.11 以來一直為關閉以避免不必要的快取前綴重寫。

## [0.8.34] - 2026-05-13

一個潤飾、終端機協定和內部清理版本。模型面向的表面是穩定的；此週期專注於前綴快取穩定性指標、更廣泛的終端機協定覆蓋、捆綁技能，以及縮減圍繞代理迴圈和 TUI 成長的巨大檔案。

### 新增

- **前綴快取穩定性追蹤。** 一個頁尾晶片顯示快取前綴在最近輪次中的穩定程度（靈感來自 Reasonix），使使用者可以在成本攀升之前發現破壞快取的編輯。
- **捆綁的 DeepSeek 原生工作流程技能。** 一組入門技能在二元檔中附帶，使全新安裝在沒有外部資源的情況下擁有可用的 `/skills` 目錄。
- **原生 Kitty + Ghostty 通知協定。** `OSC 99`（Kitty）和 `OSC 777`（Ghostty）現在與現有的桌面通知回退並列為一級支援。
- **帶有更多預設的主題選擇器。** Catppuccin、Tokyo Night、Dracula 和 Gruvbox 加入內建調色盤集；`/theme` 現在顯示即時選擇器。
- **分塊平行安全工具執行。** 引擎將無副作用的工具呼叫批次處理為分塊調度，使獨立的讀取/搜尋在一個輪次中完成，而不是逐次往返序列化。
- **取消所有 shell 作業。** 單個動作停止每個執行中的背景 shell 命令，而不是逐一取消。
- **`edit_file` 容忍印刷標點漂移。** 當精確匹配和前置空白模糊傳遞都失敗且設定 `fuzz: true` 時，工具會以智慧引號（`"`/`"` → `"`，`'`/`'` → `'`）、en/em 破折號（`–`/`—` → `-`）和不間斷空格（U+00A0 → 空格）正規化為 ASCII 來重試。捕獲瀏覽器或聊天客戶端將 Unicode 標點替換為檔案實際包含的 ASCII 的複製貼上失敗模式。

### 變更

- **`crates/tui/src/tui/ui.rs` 拆分為專注的模組。** 原本 10k 行單檔案 TUI 調度器分解為具有更清晰職責的較小模組，使審查 UI 變更不需要在腦中保留整個表面。
- **`crates/tui/src/core/engine.rs` 縮減。** 輔助叢集移至輪次迴圈和工具執行程式碼旁邊的現有 `core/engine/` 子模組目錄，使代理迴圈核心更容易端到端閱讀。
- **結構化追蹤於工具調度。** 工具進入、退出、持續時間和結果/錯誤透過 `tracing` 事件發出，使 `RUST_LOG=engine.tool_execution=debug` 產生連貫的時間線，而不是散落的臨時列印。
- **`/init` 原地更新 `AGENTS.md`**，而不是在檔案已存在時拒絕，使新增專案指引不需要手動拼接。
- **推理 token 納入成本計算**，且當會話語言環境為 `zh-Hans` 時，成本顯示自動切換為人民幣。
- **陳舊的倉庫根開發文件已移除。** `TAKEOVER_PROMPT.md`（v0.8.6 時代）、`PROMPT_ANALYSIS.md` 和多餘的 `DEPENDENCY_GRAPH.md` 不再隨發佈附帶；`docs/ARCHITECTURE.md` 保持為標準 crate 佈局參考。

### 修復

- **認證金鑰在啟動時根據已儲存的提供者檢查**，使儲存的 DeepSeek 金鑰在會話中切換提供者後不再被拒絕。
- **自動路由器對決定性本地路由跳過**，移除調度器可以直接路由的提示上的額外模型往返。
- **推理內容對不理解的通用提供者剝離**，防止指向缺乏 DeepSeek 思考語意的 OpenAI 相容閘道時出現 HTTP 400。
- **`FocusGained` 去抖動**，使發出快速焦點事件的終端機（Tabby）不再觸發重繪閃爍迴圈。
- **MCP HTTP 傳輸預設 `Accept: application/json, text/event-stream`** 並在請求之間持久化 `Mcp-Session-Id`，符合可恢復串流的規範。
- **Shell 輸出尾部在截斷時保留**，使長命令輸出的最後幾行（通常是錯誤尾部）在對話記錄摘要中存活。
- **前綴快取在修剪工具結果時保留。** 舊的副作用工具負載不再使下一輪次原本會重複使用的前綴失效。
- **審查子代理被阻止生成進一步的子代理**（#1489），保持遞迴深度有界。
- **說明覆蓋層乾淨關閉** 並在沒有陳舊畫面的情況下重繪。
- **拼音 `/skills` 別名正確調度**，使中文語言環境使用者到達相同的表面。
- **VTE 閃爍終端機預設獲得減少的動態**，以避免在錯誤處理頻繁部分重繪的終端機上發生顛簸。
- **編輯器邊框不再顯示衍生的會話標題**，保持編輯器 chrome 保留給編輯器和模式狀態。

## [0.8.33] - 2026-05-12

一個子代理和 RLM 翻新版本。模型面向的委派表面現在是會話導向而非一次性：RLM 工作透過 `rlm_open` / `rlm_eval` / `rlm_configure` / `rlm_close` 進行，子代理工作透過 `agent_open` / `agent_eval` / `agent_close` 進行，大型輸出可以停放在型別化控點後面，模型使用 `handle_read` 明確讀回。

### 新增

- **具有有界 REPL 輔助工具的持久 RLM 會話。** RLM 提示現在使用 `peek`、`search`、`chunk`、`context_meta`、`sub_query`、`sub_query_batch`、`sub_query_map`、`sub_rlm` 和 `finalize(value, confidence)`，而不是將完整的父上下文作為環境變數公開。
- **分支感知的子代理會話。** `agent_open` 支援命名會話、`fork_context` 和有界遞迴深度，使父級可以在可用時請求多個視角，同時保留有利於提示快取的前綴上下文。
- **共享 `handle_read` 儲存。** RLM 最終結果、子代理對話記錄和其他大型結構化結果可以返回帶有切片、範圍、計數和 JSONPath 投影的 `var_handle` 引用。
- **新表面的斜線命令路由。** `/rlm [N] ...` 和 `/agent [N] ...` 現在提示助理使用持久工具，而不是已移除的前景 RLM 操作。
- **Harness 友善的非互動式 exec 會話。** `deepseek exec` 現在支援 `--resume`、`--session-id`、`--continue` 和 `--output-format stream-json`，使後端包裝器（例如 ClawBench）可以保持對話狀態並每行解析一個 JSON 事件，而無需執行長期伺服器。
- **帶有 CJK 別名的 `/relay` 斜線命令**（`/接力`）。將結構化交接提示交給助理，用於跨會話的協調多輪次延續。
- **`checklist_write` 側邊欄重新命名。** 原先稱為「Plan」/「Todos」的側邊欄焦點標籤現在是「Work」——一個用於活躍檢查清單和可選計劃的面板，在所有三種模式中保持一致。
- **灰階主題。** `/theme grayscale` 和 `/set theme grayscale --save` 提供低主見的黑白調色盤，適合希望在終端機中減少品牌顏色的使用者。

### 變更

- **提示和文件現在僅教導新工具名稱。** 舊版 RLM/子代理輔助工具在需要持久對話記錄相容性的地方保留在內部，但登錄表公開會話工具。
- **大型或嘈雜的工具結果更容易保持在上下文之外。** 工具輸出摘要、子代理結果和對話記錄快照現在在需要原始細節時引導模型使用 `handle_read`。
- **工具表面煙霧指引是明確的。** 發佈檢查現在記錄 `handle_read`、持久 RLM 工具和持久子代理工具的確切版本命令和登錄表名稱搜尋。
- **README 致謝部分擴展。** 專案感謝 OpenWarp 和 Open Design 在終端機代理和設計導向工作流程方面的支援和協作。
- **淺色主題調整為更平靜的對比度。** 畫布、面板、層疊、邊框和選擇 token 現在分隔表面而不帶有褪色的白底白字感覺。
- **會話選擇器以歷史記錄為先。** `/sessions` 和 `Ctrl+R` 現在在左側顯示選定會話的完整歷史記錄，在右側顯示會話列表；數字鍵 `1`-`9` 開啟可見的會話歷史記錄，`PgUp` / `PgDn` 捲動該歷史記錄，`Enter` 仍然恢復。
- **前景 RLM 操作已移除。** 舊的 `Op::Rlm` 路徑及其 `handle_rlm` 引擎方法已消失；所有 RLM 工作現在透過持久會話工具進行。
- **陳舊的競爭分析文件已移除。** 舊的跨代理矩陣已成為工具名稱的不可靠清單，而非有用的發佈指引。

### 修復

- **本地/自訂端點在認證可選時保持無提示。** 調度器不再為 SGLang、vLLM、Ollama 或回送自訂 URL 讀取密鑰儲存，除非明確要求 API 金鑰認證，且直接 TUI 預設將回送模型端點視為無金鑰。這避免了在使用者將應用指向本地 OpenAI 相容伺服器時出現 macOS 鑰匙圈提示和陳舊的 DeepSeek 金鑰。
- **對話記錄瀏覽在調整大小期間保持位置。** 如果使用者正在閱讀較舊的聊天歷史記錄，終端機調整大小事件保留目前的對話記錄位置，而不是跳回即時尾部；捲動條和跳到最新的便利功能現在遵循活躍主題。
- **回溯預覽在選定輪次附近開啟。** 按兩次 Esc 不再在最舊的對話行開啟即時對話記錄預覽；反白顯示的最近使用者輪次被固定到視圖中，變更回溯目標重新選擇僅固定該選擇。
- **已完成的思考不再偽裝成提示文字。** 折疊的已完成推理現在僅內聯顯示明確的 `Summary:` 內容；原始推理仍然可透過 Ctrl+O/對話記錄取得，而不是在主流程中作為助理自言自語出現。當 Ctrl+O 從推理區塊開始時，它開啟完整會話推理時間線，而不是單個孤立區塊。
- **對話記錄選擇在代理串流時繼續工作。** 載入狀態滑鼠過濾器現在丟棄惰性移動事件，但允許活躍對話記錄和捲動條拖曳繼續（在 v0.8.32 中報告為已知問題）。
- **空白編輯器方向鍵捲動感覺不那麼跳躍。** 當設定為捲動對話記錄時，普通上/下現在以類似滾輪的小步移動，而不是單行跳動。
- **滑鼠和觸控板捲動在長日誌中感覺不那麼粘。** 快速的同方向對話記錄捲動現在獲得有界加速，而方向變更重置為精確的單行移動。
- **RLM 煙霧測試小問題修復。** `rlm_eval` 現在將 `content` 綁定為 `_context` 的便利別名，容忍子查詢輔助工具上常見的 `timeout_secs` 關鍵字猜測同時保留會話級逾時政策，並將 JSON 可序列化的 `finalize(...)` 值存儲為 JSON 控點，使 `handle_read` 可以直接投影它們。
- **RLM REPL 使用共享的 Python 解析器。** RLM 啟動現在嘗試 `python3`、`python` 和 `py -3`，與程式碼執行使用的相依解析器匹配，避免在 `python3` 不存在的 Windows 上失敗（從 PR #1540 收集）。
- **會話標題和歷史記錄預覽隱藏中繼資料噪音。** 已儲存的會話標題和選擇器歷史記錄剝離前導的 `<turn_meta>` 封套和思考標籤區塊，使歷史對話讀起來像使用者可見的聊天而不是提示管道（從 PR #1510 收集）。
- **伴隨二元檔版本煙霧是明確的。** `deepseek-tui --version` 現在報告 `deepseek-tui` 二元檔名稱而不是調度器標籤。
- **視覺路徑邊界測試是平台原生的。** 絕對路徑拒絕煙霧在 Windows 上使用 Windows 絕對路徑，在其他地方使用 `/etc/hosts`（從 PR #1526 收集）。
- **工具小問題：** `file_search` 具有更安全的預設排除和明確的 `exclude` 選項；`grep_files` 將單行上下文返回為字串；`fetch_url` 可以投影 JSON 欄位並返回標頭；`edit_file` 可以選擇加入前導縮排模糊匹配；`exec_shell` 可以按時間順序合併 stdout/stderr；`revert_turn` 拒絕無作用的快照邊界。
- **CLI reasoning-effort 在非自動 exec 路由上生效**（PR #1511 來自 **@h3c-hexin**）。`deepseek -p "..." --reasoning-effort high` 現在正確應用旗標，而不是退回設定檔預設值。
- **Edit-file 替換邊界已釐清**（PR #1516）。工具描述和錯誤訊息現在明確指出 `edit_file` 用於一個檔案中的一個明確替換。
- **Pandoc 輸出在探測前驗證**（PR #1523）。產生空白或無效輸出的二元格式轉換現在顯示明確的錯誤，而不是令人困惑的 pandoc 堆疊追蹤。
- **執行中的輪次可以被引導和重繪**（PR #1533、#1537）。活躍輪次期間的編輯器輸入不再停滯；TUI 在代理串流時重繪對話記錄。
- **Tasks 和 Activity Detail 在負載下更平靜。** Tasks 面板現在保持即時/背景/最近活動不重複計數相同的 shell 或 RLM 工作，分組重複的讀取/搜尋/檢查清單噪音，並保持失敗、狀態、命令摘要和持續時間可見。Ctrl+O 現在為選定/即時/最近工具工作開啟 Activity Detail，為思考區塊開啟推理時間線，而 Alt+V 仍然是直接的工具細節分頁器；閒置頁尾現在為可見活動公告該分割。
- **npm 重試在首次失敗時顯示逾時提示**（PR #1538）。在慢速代理後面的安裝現在看到清晰的「重試中」訊息，而不是沉默卡住。
- **問題範本改進**（PR #1525 來自 **@reidliu41**）。錯誤和功能請求範本對新貢獻者來說更清晰、更容易使用。

## [0.8.32] - 2026-05-12

一個「更有用工具」的版本。v0.8.31 使工具表面在每個主機上可靠；v0.8.32 擴展了它。核心是每個新貢獻者問的問題：「模型實際上有什麼可用的？」——現在答案更接近「你從 shell 會使用的所有東西，包括真實世界使用的文件格式。」五個新工具（`pdf-extract` 交換、`js_execution`、`pandoc_convert`、`image_ocr`、`image_analyze`），十個社群 PR 收集針對模型協定錯誤（vLLM 思考）和 UX 小問題（Windows VSCode Shift+Enter、提及截斷分割 CJK 碼點、審批模態隱藏對話記錄），以及對超過 2 GB 非排除內容的工作區的快照自停用，使首輪 `git add -A` 不再在數百 GB 的專案目錄上卡住 TUI。

### 效能

- **將 `instructions = [...]`、使用者記憶體和會話目標移至提示的揮發性內容邊界之下，使 DeepSeek 的 KV 前綴快取在會話中編輯時存活**（從 PR #1345 由 **@Duducoco** 收集）。在此變更之前，每個工作區的 `instructions` 區塊、使用者記憶體檔案（`/memory`）和目前會話目標（`/goal`）在系統提示中的位置 2.5 渲染——在快取命中的靜態前綴層內部。對這些檔案的任何編輯（或任何 `# foo` 快速新增到記憶體）從該位元組開始破壞快取前綴，迫使下一個輪次重新對靜態層的其餘部分進行 token 化。將它們重新定位到位置 6（緊接在前一個會話交接區塊上方）意味著快取命中覆蓋整個靜態前綴——模式、專案上下文、環境、技能、上下文管理、壓縮範本——無論使用者多頻繁編輯其記憶體檔案。技能、上下文管理和壓縮範本保持在它們所屬的始終可快取的靜態層中。
