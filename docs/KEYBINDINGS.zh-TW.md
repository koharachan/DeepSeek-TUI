# 快速鍵

[English](KEYBINDINGS.md)

這是 TUI 識別的每個鍵盤快速鍵的權威目錄。快速鍵按**上下文**分組 — 它們生效的焦點或模態狀態。列在「撰寫器」下的快速鍵僅在撰寫器獲得焦點時生效；列在「轉錄」下的快速鍵僅在轉錄獲得焦點時生效；以此類推。

快速鍵目前尚未開放使用者設定 — 追蹤中以供未來版本使用 (#436, #437)。本文件是未來設定檔案覆蓋將命名的合約。

## 全域（任何上下文）

| 組合鍵                | 操作                                                        |
|----------------------|---------------------------------------------------------------|
| `F1` 或 `Ctrl-/`     | 切換說明覆蓋層                                       |
| `Ctrl-K`             | 開啟命令面板（斜線命令尋找器）                |
| `Ctrl-C`             | 取消目前回合 / 關閉模態框 / 預備後確認退出    |
| `Ctrl-D`             | 退出（僅在撰寫器為空時）                         |
| `Tab`                | 循環 TUI 模式：Plan → Agent → YOLO → Plan                     |
| `Shift-Tab`          | 循環推理力度：off → high → max → off                 |
| `Ctrl-R`             | 開啟繼續工作階段選擇器                                 |
| `Ctrl-L`             | 重新整理 / 清除螢幕                                     |
| `Ctrl-O`             | 開啟選定/正在執行/最近工具工作的活動詳細資訊，或當撰寫器為空時開啟思維區塊的完整推理時間軸 |
| `Ctrl-Shift-E` / `Cmd-Shift-E` | 切換檔案樹側邊欄                          |
| `Alt-!` / `Alt-@` / `Alt-#` / `Alt-$` / `Alt-0` | 焦點：工作 / 任務 / 代理 / 上下文 / 自動側邊欄 |
| `Ctrl-Alt-0`         | 隱藏右側側邊欄                                          |
| `Esc`                | 關閉最上層模態框 · 取消斜線選單 · 關閉提示訊息        |

## 撰寫器

編輯您即將傳送的訊息。

| 組合鍵                       | 操作                                                  |
|-----------------------------|---------------------------------------------------------|
| `Enter`                     | 傳送訊息（或執行斜線命令）             |
| `Alt-Enter` / `Ctrl-J`      | 插入換行而不傳送                        |
| `Ctrl-U`                    | 刪除至行首                                 |
| `Ctrl-W`                    | 刪除前一個字詞                                    |
| `Ctrl-A` / `Home`           | 移至行首                                   |
| `Ctrl-E` / `End`            | 移至行尾                                     |
| `Ctrl-←` / `Alt-←`          | 向後移動一個字詞                                  |
| `Ctrl-→` / `Alt-→`          | 向前移動一個字詞                                   |
| `Ctrl-V` / `Cmd-V`          | 從剪貼簿貼上（也自動處理 bracketed-paste）|
| `Ctrl-Y`                    | 從刪除緩衝區中拉出（貼上）                           |
| `↑` / `↓`                   | 循環撰寫器歷史記錄（也用於選取彈出/附件項目） |
| `Ctrl-P` / `Ctrl-N`         | 循環撰寫器歷史記錄（替代方式）                     |
| `Ctrl-S`                    | 暫存目前草稿（`/stash list`、`/stash pop` 恢復） |
| `Alt-R`                    | 搜尋提示歷史記錄（Alt-R 退出）                  |
| `Tab`                       | 斜線命令 / `@`-提及完成（支援彈出視窗）    |
| `Ctrl-O`                    | 在撰寫器獲得焦點時，以外部編輯器開啟撰寫器草稿 |

### `@` 提及

輸入 `@<部分名稱>` 來開啟檔案提及彈出視窗。`↑`/`↓` 循環項目，`Tab` 或 `Enter` 接受。`Esc` 隱藏彈出視窗。自 v0.8.10 起 (#441)，完成項目會按提及頻率重新排序 — 您經常 + 最近提及的檔案會排在最前面。

### `#` 快速新增（記憶體）

當 `[memory] enabled = true` 時，輸入 `# foo` 並按 `Enter` 會將 `foo` 作為附帶時間戳記的項目符號新增到您的記憶體檔案，而**不**傳送回合。請參閱 `docs/MEMORY.md`。

|----------------------|-----------------------------------------------------|
| `↑` / `↓` / `j` / `k`| 捲動一行（v0.8.13+：當撰寫器為空時，裸方向鍵也可捲動） |
| `PgUp` / `PgDn`      | 捲動一頁                                    |
| `Home` / `g`         | 跳至頂端                                         |
| `End` / `G`          | 跳至底部                                     |
| `Esc`                | 將焦點返回撰寫器                           |
| `y`                  | 將選取區域拉出至剪貼簿                  |
| `v`                  | 開始 / 擴展視覺選取                    |
| `o`                  | 開啟游標下的 URL（支援 OSC 8 的終端機）    |

## 側邊欄（當側邊欄獲得焦點時）

| 組合鍵                | 操作                                              |
|----------------------|-----------------------------------------------------|
| `↑` / `↓` / `j` / `k`| 移動選取                                     |
| `Enter`              | 啟用選定的項目（開啟 / 焦點 / 取消） |
| `Tab`                | 循環至下一個側邊欄面板（Work → Tasks → Agents → Context） |
| `Esc`                | 將焦點返回撰寫器                           |

## 斜線命令面板（`Ctrl-K` 或輸入 `/` 後）

| 組合鍵                | 操作                                              |
|----------------------|-----------------------------------------------------|
| `↑` / `↓`            | 移動選取                                     |
| `Enter` / `Tab`      | 執行 / 完成高亮顯示的命令             |
| `Esc`                | 關閉面板                                     |

## 工作階段選擇器（`Ctrl-R` 或 `/sessions`）

| 組合鍵                | 操作                                              |
|----------------------|-----------------------------------------------------|
| `↑` / `↓` / `j` / `k`| 在工作階段清單中移動選取                 |
| `1`-`9`              | 在該清單位置開啟可見的工作階段歷史記錄 |
| `PgUp` / `PgDn`      | 翻頁歷史記錄面板                              |
| `Enter`              | 繼續選定的工作階段                        |
| `/`                  | 搜尋工作階段                                    |
| `s`                  | 循環排序順序                                   |
| `a`                  | 切換目前工作區範圍 vs 所有工作區   |
| `d`                  | 確認後刪除選定的工作階段         |
| `Esc` / `q`          | 關閉選擇器                                   |

## 審核模態框（當工具請求審核時）

| 組合鍵                | 操作                                              |
|----------------------|-----------------------------------------------------|
| `y` / `Y`            | 核准一次                                        |
| `a` / `A`            | 全部核准（自動核准後續呼叫）        |
| `n` / `N` / `Esc`    | 拒絕                                                |
| `e`                  | 在執行前編輯已核准的輸入              |

## 入門導引（首次執行流程）

| 組合鍵                | 操作                                              |
|----------------------|-----------------------------------------------------|
| `Enter`              | 前往下一步（歡迎 → 語言 → API → …） |
| `Esc`                | 返回上一個畫面                                |
| `1`–`5`              | 選擇語言（語言步驟）                    |
| `y` / `Y`            | 信任工作區（信任步驟）                   |
| `n` / `N`            | 跳過信任提示                              |

## v0.8.29 審計記錄

- **`Shift+Enter` / `Alt+Enter` 換行現在在 VSCode on Windows 上可正常使用 (#1359)。** crossterm 的 `PushKeyboardEnhancementFlags` 命令在 Windows 上無條件返回 `Unsupported`（`is_ansi_code_supported() == false`），因此 Kitty 鍵盤協定逸出碼永遠不會寫入終端機。沒有它，VSCode 的 xterm.js 保持在舊版模式，其中 `Shift+Enter` 與普通 `Enter` 無法區分，導致撰寫器傳送訊息而非插入換行。此修正直接在 Windows 上寫入 push/pop 逸出碼（`\x1b[>1u` / `\x1b[<1u`），繞過 crossterm 的能力門檻。VSCode 整合終端機和 Windows Terminal ≥1.17 都支援 Kitty 鍵盤協定；不支援這些序列的終端機會靜默地丟棄它們。

## v0.8.13 審計記錄

- **Ctrl-S 是暫存，而非歷史記錄搜尋。** 在此修訂版中修正 — `Alt-R` 是歷史記錄搜尋。
- **移除了不存在的 `Alt+Up`。**「編輯最後排入佇列的訊息」快速鍵列在 README 中，但在按鍵分派程式碼中從未存在。
- **當撰寫器為空時，裸上/下方向鍵可捲動轉錄內容 (v0.8.13)。** 此前 `should_scroll_with_arrows` 閘門被硬編碼為 false，意味著裸方向鍵始終導航撰寫器歷史記錄，即使撰寫器為空。虛擬終端機（Ghostty、Codex、Kitty-protocol）的使用者尤其受影響，因為他們無法使用 Cmd+Up / Alt+Up 快捷鍵。
- **可設定的按鍵對應 (#436) 和 `tui.toml` (#437) 仍然延期。** `TuiPrefs` 結構和載入器存在於 `settings.rs` 中，但尚未在啟動時接線。將允許 `~/.deepseek/tui.toml` 覆蓋個別項目的命名快速鍵登錄表仍在等待中。
- **沒有發現其他損壞的快速鍵。** 上面列出的每個其他組合鍵都對應到 `crates/tui/src/tui/ui.rs`（按鍵事件分派）或 `crates/tui/src/tui/app.rs`（模式 + 狀態轉換）中的有效處理常式。
